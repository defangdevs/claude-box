▣ claude-box

A private, sandboxed Claude Code or Codex agent for you and your team — launch it in your own AWS account with one click.

Launch on AWS View on GitHub

What it is

A ready-to-use coding agent you open right in your browser. Set it up once with a single click; it keeps running on its own.

Locked down by default

Each agent runs in its own restricted account and can't touch the rest of the machine — it only gets the specific extra powers you choose to grant.

Works in your browser

A full terminal, right in your browser over a secure connection. Open your link and you're in — nothing to install.

Choose your agent

Pick Claude Code or Codex when you launch. The same NixOS module, terminal, and sandboxing model support both.

Keeps your keys safe

Any API keys or tokens you add stay private — kept out of shared settings and never baked into saved images.

Low cost

Runs lean and skips an optional AWS network fee by default, so a small box typically costs only a few dollars a month. Shut it down anytime to stop charges.

1-click AWS launch

Sets up one ready-to-go box in your own AWS account with the browser terminal already running. It creates everything it needs, so there's nothing to configure beforehand — just pick a region and go.

Your current network can't reach IPv6-only servers, so the Launch buttons above now include a public IPv4 address — without it, the box wouldn't be reachable from here. Don't want the extra cost? Uncheck PublicIpv4 on the CloudFormation form — just make sure you'll connect from a network that does have IPv6.
Set a password you'll remember. On the AWS form, type any 16–64 character password into WebPassword — that's what you'll use to log into the browser terminal. AWS masks it with dots after you type it and never shows it again, so save it somewhere before you launch. Choose Claude Code or Codex, pick a size, rename the stack if you want a friendlier Claude app label, and launch. The box takes about 2–3 minutes to start; your browser link then appears under Outputs in the AWS console — open it, enter the password, finish the selected agent's one-time sign-in, and you're in. (Claude Code stacks can drive the box from the Claude desktop / mobile app via Remote Control after the first browser login — the stack name becomes the session name.)
The optional ~$3.60/month, explained. By default the box skips the one thing AWS charges for on the network side — a public IPv4 address — so connectivity costs $0/month, using a free IPv6 address instead. That works fine if the networks you connect from support IPv6 (most home connections do). Checking PublicIpv4 on the AWS form adds a stable IPv4 for ~$3.60/month; it's worth paying for if you'll open the box from offices or public Wi-Fi (which often lack IPv6), or if the box itself needs to reach IPv4-only sites — notably github.com, which an IPv6-only box can't clone from or push to. We check your browser on this page and switch the Launch buttons to IPv4 automatically if we can't reach the IPv6 internet from here.
